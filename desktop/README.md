# Desktop packaging (contributor-only)

This folder contains the PyInstaller spec, build script, and macOS code-signing
assets used to **build a packaged ViralMint app from this OSS source**. It
exists so contributors can smoke-test their changes inside an actual `.app`
bundle before opening a PR — not as the user-facing install path.

> **End users should follow the main [README](../README.md): `pip install -r requirements.txt && python run.py`.** Prebuilt desktop installers (with extra paid features like the Tools page, AI Music tab, and Visual Style preset) are sold separately at [viralmint.net](https://viralmint.net).

## What you get

A single-binary headless app that:

- Starts the FastAPI backend on `127.0.0.1:16888`.
- Opens your default browser at that URL once the port is up.
- Writes user data (`viralmint.db`, `storage/`, `logs/`, `.env`) under `~/ViralMint/` (override with `VIRALMINT_DATA_DIR`).
- Has no system tray, no animated UI, no auto-update — the browser is the UI.

## Prerequisites

- macOS 12+ (or Linux x86_64, or Windows 10+ with MSYS-style bash)
- Python 3.11+ in a venv at the repo root (`python -m venv venv`)
- Node 18+
- `npm`, `bash`, `unzip`
- macOS only: Xcode Command Line Tools (for `codesign`, `iconutil`, `hdiutil`, `sips`)

## Build

From the repo root:

```bash
PYTHON_BIN=./venv/bin/python VIRALMINT_VERSION=0.1.0-test \
  bash desktop/scripts/build-app.sh
```

What it does:

1. `pip install -r requirements.txt` + PyInstaller into `$PYTHON_BIN`.
2. Builds the frontend (`npm install && npm run build`) → `frontend/dist/`.
3. Vendors a static `ffprobe` binary → `desktop/scripts/vendor/`.
4. Generates `icon.icns` (macOS) or `icon.ico` (Windows) from `desktop/build/icon.png`.
5. Runs PyInstaller on `desktop/scripts/viralmint.spec` → `dist/ViralMint.app` (macOS) or `dist/ViralMint/` (Linux/Windows).
6. Packages → `desktop/release/ViralMint-<version>-(mac.dmg|mac.zip|linux.tar.gz|win.zip)`.

Total time on a clean checkout: **~10–15 minutes** (the PyInstaller step is the long pole).

## Skip flags for iteration

Re-run after a code-only change:

```bash
SKIP_FRONTEND=1 SKIP_FFPROBE=1 \
  PYTHON_BIN=./venv/bin/python VIRALMINT_VERSION=0.1.0-test \
  bash desktop/scripts/build-app.sh
```

`SKIP_FRONTEND=1` reuses the existing `frontend/dist/`. `SKIP_FFPROBE=1` reuses the vendored `ffprobe`. With both set, only PyInstaller runs (~5–7 min).

## Smoke testing the build

Don't drag straight to `/Applications` — test from `dist/` first so a bad build doesn't pollute your system:

```bash
# Quick check that the bundled binary boots cleanly
BROWSER=/usr/bin/true ./dist/ViralMint.app/Contents/MacOS/ViralMint &
sleep 8
curl -sS http://127.0.0.1:16888/api/config/tts_providers | jq .
pkill -f 'ViralMint.app/Contents/MacOS/ViralMint'
```

If that works, mount the DMG (`open desktop/release/ViralMint-*.dmg`) and drag the `.app` to `/Applications`.

When something is wrong, two places to look:

- `~/ViralMint/logs/app.log` — startup banner + uvicorn-launch errors.
- macOS unified log: `log show --predicate 'processImagePath CONTAINS "ViralMint"' --last 5m`.

## Code signing & notarization (optional)

Without any env vars, the build ad-hoc-signs the `.app` so it runs on the build machine; Gatekeeper warns other users on first launch. For a properly signed + notarized build, export before invoking the script:

| Var | Purpose |
| --- | --- |
| `CSC_LINK` | Base64-encoded macOS Developer ID `.p12` |
| `CSC_KEY_PASSWORD` | Password for the `.p12` |
| `APPLE_ID` | Apple ID for notarization |
| `APPLE_APP_SPECIFIC_PASSWORD` | App-specific password |
| `APPLE_TEAM_ID` | Apple Developer team ID |

Notarization is a separate Apple service call and adds 5–30 min to the build; only do it for releases you actually intend to distribute.

## What this packaging does NOT include

- **System-tray launcher.** The OSS bundle is headless — backend + browser. The paid desktop installer at viralmint.net wraps this with a tray launcher, auto-update, and signed binaries.
- **Auto-update of the bundled `yt-dlp`.** PyInstaller bundles aren't pip-managed; the in-app updater is disabled in frozen mode (see `backend/services/ytdlp_service.py`). Re-build to refresh.
- **Whisper model bundling.** First transcription downloads the chosen model into `~/ViralMint/whisper-cache/`.
- **Windows MSI / Linux .deb / Linux AppImage.** The script produces a `.zip` or `.tar.gz` on those platforms — runnable, but not a polished installer. Add WiX/AppImage steps if you want them.

## Layout

```
desktop/
├── README.md                       # this file
├── build/
│   ├── entitlements.mac.plist      # hardened-runtime entitlements
│   ├── icon.icns                   # generated by build-app.sh from icon.png
│   └── icon.png                    # source icon (1024×1024)
├── release/                        # build output (gitignored)
└── scripts/
    ├── build-app.sh                # entry point — runs the whole pipeline
    ├── fetch-ffprobe.sh            # downloads platform-specific ffprobe
    ├── vendor/                     # ffprobe lands here (gitignored)
    └── viralmint.spec              # PyInstaller spec — single-bundle layout
```

The PyInstaller entry is `desktop_app.py` at the repo root, not `run.py` — `run.py` is the dev-mode entry that builds the frontend on the fly and assumes Python + Node are installed.
